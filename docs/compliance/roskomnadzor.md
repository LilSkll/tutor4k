# Соответствие 152-ФЗ и уведомление Роскомнадзора

> **Важно:** этот документ — практический чеклист для оператора сервиса Spanish with Pavel.  
> Перед подачей уведомления проконсультируйтесь с юристом по персональным данным.

| Требование | Реализация |
|---|---|

## 2. Нужно ли уведомлять Роскомнадзор?

По **152-ФЗ** оператор обязан уведомить Роскомнадзор **до начала обработки** персональных данных, **если** обработка не подпадает под исключения (ст. 22).

### Типичные исключения (упрощённо)

Уведомление **может не требоваться**, если одновременно:

- обработка только для исполнения договора с субъектом (оказание образовательного сервиса);
- данные получены **только от самого пользователя** (регистрация, профиль, прогресс);
- нет трансграничной передачи в страны без адекватной защиты **без** дополнительных мер;
- нет обработки специальных категорий ПДн (здоровье, биометрия и т.д.);
- нет массовой публичной публикации ПДн.

### Когда уведомление **скорее всего нужно**

- вы обрабатываете ПДн граждан РФ **и** используете зарубежные облака (Supabase/Vercel/AI API) с передачей за рубеж;
- планируете маркетинговые рассылки на основании согласия;
- обрабатываете данные несовершеннолетних без согласия родителей;
- оператор — ИП/ООО с сотрудниками, обрабатывающими ПДн от имени компании.

**Для индивидуального оператора (физлицо/ИП) с SaaS и трансграничной передачей** на практике уведомление подают в большинстве случаев — особенно если серверы и AI-провайдеры за пределами РФ.

## 3. Пошаговая подача уведомления

1. Зарегистрируйтесь на [pd.rkn.gov.ru](https://pd.rkn.gov.ru) (личный кабинет оператора ПДн).
2. Подготовьте сведения:
   - **Оператор:** ФИО / наименование ИП, ИНН (если есть), адрес, email `legal@spanishwithpavel.com`
   - **Цель обработки:** предоставление онлайн-сервиса изучения языков
   - **Категории субъектов:** пользователи сервиса (ученики)
   - **Категории ПДн:** ФИО (имя), email, учебный прогресс, тексты диалогов с AI
   - **Правовое основание:** согласие + договор (пользовательское соглашение)
   - **Меры защиты:** HTTPS, Supabase RLS, хеширование паролей, ограничение доступа
   - **Трансграничная передача:** указать Supabase (регион проекта), Vercel, Groq/Gemini — со ссылкой на DPA/SCC
   - **Срок хранения:** до удаления аккаунта + до 90 дней в резервных копиях
   - **Место размещения политики:** `https://spanishwithpavel.com/privacy`
3. Подайте уведомление **до** активного привлечения пользователей из РФ (если ещё не подали).
4. Сохраните PDF-подтверждение из личного кабинета.

## 4. GDPR / UK (преподаватели UK и Испании)

- Англоязычная политика: `/privacy?lang=en`
- Terms: `/terms?lang=en`
- Пользователи EU/UK имеют право на экспорт и удаление (реализовано в Settings).
- При работе с UK: при необходимости зарегистрируйтесь в ICO (плата £40–£60/год для микроорганизаций) — [ico.org.uk](https://ico.org.uk).
- Для Испании (AEPD): как правило, регистрация не требуется для микропредприятий без масштабной обработки, но Privacy Policy и DPA с процессорами обязательны.
